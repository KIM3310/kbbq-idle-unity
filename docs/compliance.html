Compliance & Quality

This page documents content-quality and ad-separation standards used by the KBBQ Idle WebGL service.

Playable-First Layout

  • The main gameplay canvas remains the primary content above policy and sponsored sections.
  • Operational controls (build check/start/fullscreen) are clearly separated from ads.
  • Error states are displayed to users with actionable status messaging.

Ad Separation Policy

  • Sponsored blocks are labeled and visually separated from gameplay controls.
  • Ads are disabled unless valid publisher and slot IDs are configured.
  • No hidden, auto-overlay, or deceptive ad placement is used.

Transparency Requirements

  • Privacy, Terms, Contact, and Compliance links are always available in top navigation.
  • Deployment output mapping and build status are visible on the Play page.
  • Known service changes are tracked in repository history.

Quality Controls

  • WebGL preflight validation checks required build artifacts before game start.
  • Unity data validator and CI smoke checks run before release.
  • Runtime build-manifest metadata is generated during docs build for deterministic loading.
